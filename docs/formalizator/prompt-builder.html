Конструктор запроса к моделированию актива

Вкладки по порядку работы. Файл data/actives.xlsx, лист Справочник, столбцы A–D (№ п/п, ДО, Месторождение, Пласт) → python scripts/actives_to_json.py → actives_tree.json (в конструкторе — только ДО и месторождения). Экспорт build_excel.py: справочник активов — лист Справочник_ДО.

Запуск: python -m http.server 8080 из корня проекта.

Выбор активов сценария

Каждое ДО — отдельный блок ниже; месторождения только из строк этого блока. После смены actives_tree.json снова нажмите «Загрузить всё с сервера» или перезагрузите страницу с обходом кэша (Ctrl+F5).

Быстрые пресеты

Можно выбрать готовую постановку или пропустить этот шаг и листать ниже, чтобы собрать свой промпт вручную (база, горизонт, цели, ограничения, рычаги).

Условия сценария

Где нужно несколько вариантов — отметьте несколько галочек (например две базы или две цели). Пустой блок значит «модель уточняет сама».

Текст промпта

Текст можно править в поле. «Обновить» пересоберёт его из конструктора (ручные правки будут заменены). Зелёным выделены типовые формулировки, белым — выбранные актив, горизонт, цели, ограничения и рычаги.

Сначала на вкладке «Конструктор промпта» нажмите «Подтвердить промпт → открыть анализ причин». До этого разбор не строится.

Элементы промпта — как в поле промпта. Кратко: контекст и типовые причины отклонения.
