Имена секретов должны совпадать **точно** (регистр важен).

### GitHub Actions SSH key setup

Это **отдельная** пара ключей только для входа GitHub Actions на ваш сервер по SSH. Её не нужно путать с *deploy key* репозитория (тот ключ нужен серверу, чтобы делать `git pull` с GitHub).

**1. На своём компьютере** (не на сервере) сгенерируйте пару без парольной фразы:

```bash
ssh-keygen -t ed25519 -f ./gha_deploy -N '' -C "github-actions-deploy"
```

Появятся два файла: `gha_deploy` (приватный) и `gha_deploy.pub` (публичный).

**2. На сервере** под пользователем деплоя (`yc-user`) добавьте **одну строку** из `gha_deploy.pub` в файл авторизации:

```bash
# на сервере, или так:
cat gha_deploy.pub >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
```

Удобно скопировать pubkey с ПК: `ssh-copy-id -i gha_deploy.pub yc-user@158.160.149.143` (если уже есть другой вход на сервер).

**3. Проверка с ПК** — должно пускать **без пароля**:

```bash
ssh -i ./gha_deploy yc-user@158.160.149.143
```

**4. В GitHub** создайте секрет **`SSH_PRIVATE_KEY`**: откройте файл **`gha_deploy`** (именно приватный, не `.pub`) в текстовом редакторе, скопируйте **всё** от первой строки `-----BEGIN OPENSSH PRIVATE KEY-----` или `-----BEGIN ... PRIVATE KEY-----` до последней `-----END ...-----` включительно, вставьте в поле значения секрета. Пробелы и переносы строк сохраняются.

**5. Файлы `gha_deploy` и `gha_deploy.pub`** храните только у себя (не коммить в репозиторий). При утечке приватного ключа — удалите соответствующую строку из `authorized_keys` на сервере и сгенерируйте новую пару.

## Безопасность

- Не коммить в git файлы с реальным `SSH_PRIVATE_KEY`, PAT или ключами API.
- При утечке ключа — отозвать в GitHub / на сервере удалить строку из `authorized_keys` и сгенерировать новую пару.
