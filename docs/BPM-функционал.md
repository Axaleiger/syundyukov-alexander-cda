# Функционал Планирования (BPM)

Документ описывает возможности вкладки **Планирование** и её связь с Конфигуратором систем и образцовым сценарием.

## Назначение

Вкладка «Планирование» предназначена для настройки сценариев в виде доски: этапы, карточки задач, исполнители, системы, входы/выходы. Данные синхронизируются с вкладкой **Конфигуратор систем**: при изменении доски генерируется Mermaid-схема (Код/Схема), при переключении на Конфигуратор отображается актуальный граф.

## Источники данных и эталон

- **Логика и формат:** соответствуют Streamlit-приложению из `public/app.py.txt`.
- **Образцовый сценарий:** `public/hantos.xlsx` — загружается автоматически при открытии Планирования для актива «Хантос» (Зимнее).
- **Списки:** персонал и системы заданы в `src/data/bpmData.js` (PERSONNEL, SYSTEMS_LIST), по структуре как в app.py.

## Перечень функций

### Доска

- **Этапы** — колонки с названиями (например: Геологоразведка и работа с РБ, Разработка, Планирование и обустройство, Бурение и ВСР, Добыча).
- **Карточки** — задачи внутри этапа: ID, название, исполнитель, согласующий, срок сдачи, статус, дата создания.
- **Записи по карточке** — используемые системы, входные данные, выходные данные (несколько строк на карточку).
- Добавление этапа, добавление задачи в этап.
- Редактирование названия этапа, удаление этапа.
- Редактирование карточки (форма с полями и списком систем/вход/выход), удаление карточки.
- Перетаскивание карточек между этапами и этапов по порядку.

### Excel

- **Загрузить из Excel** — загрузка доски из файла. Ожидаемые колонки: «Этап Название», «Карточка ID», «Карточка Название», «Исполнитель», «Согласующий», «Срок сдачи», «Статус», «Дата создания», «Используемые системы», «Входные данные», «Выходные данные». Валидация при загрузке.
- **Выгрузить доску** — экспорт текущей доски в Excel в том же формате.
- **Шаблон** — скачивание пустого шаблона с заголовками колонок.

### Поиск и фильтры

- **Поиск по доске** — фильтрация карточек по названию, ID, исполнителю, согласующему, статусу, названию системы.
- **Фильтры** — панель с фильтром по статусу (в работе, завершен, ошибка, пауза, все).

### Режимы отображения

- **Подробный вид** — развёрнутые карточки с полями и списком систем.
- **Упрощенный вид** — свёрнутые карточки (заголовок + раскрытие по клику).

### Граф и аналитика

- **Рассчитать** — переход к виду «граф + аналитика»: визуализация этапов и карточек (vis-network), подсчёт узлов/рёбер, узлы по типам (Этап, Карточка, Система), среднее связей по типу, топ узлов по степени.

### Синхрон с Конфигуратором

- При любом изменении этапов или карточек вызывается callback `onBoardChange(stages, tasks)`.
- В приложении по этим данным строится Mermaid-флоучарт (функция `bpmToMermaid` в `src/lib/bpmToMermaid.js`) и обновляется общий код схемы (вкладка «Код» и «Схема» в Конфигураторе систем).

## Пресеты по активам

- **hantos** — ООО «Газпромнефть-Хантос» / Зимнее: при первом открытии подгружается `hantos.xlsx`.
- **nng** — ННГ / Новогоднее.
- **mgn** — Мегион / Аганское.

Идентификатор доски выбирается по выбранному объекту на главной странице (карта).
